# Шпаргалка по Python для этого проекта

## 1. Что такое декоратор

Декоратор — это функция, которая принимает другую функцию, добавляет ей новое поведение и возвращает новую функцию.

Идея:

```python
def decorator(func):
    def wrapper(*args, **kwargs):
        print("До вызова функции")
        result = func(*args, **kwargs)
        print("После вызова функции")
        return result
    return wrapper
```

Использование:

```python
@decorator
def say_hello():
    print("Hello")
```

Это почти то же самое, что:

```python
def say_hello():
    print("Hello")

say_hello = decorator(say_hello)
```

### Как это работает в твоем проекте

В файле `modules/decorators.py` есть декоратор `log_execution_time`.

Он:

- принимает функцию `func`
- внутри создает `wrapper`
- перед вызовом функции запоминает время начала
- вызывает исходную функцию
- после вызова считает, сколько времени прошло
- возвращает не только результат, но и время выполнения

То есть после декорирования функция возвращает:

```python
result, elapsed_time
```

Пример из твоего проекта:

```python
@log_execution_time
def calculate_average_of_even_numbers(numbers):
    ...
```

Теперь эта функция возвращает не просто словарь, а кортеж:

```python
(result, elapsed_time)
```

## 2. Что такое генератор

Генератор — это способ выдавать значения по одному, а не создавать все сразу.

Главевое слово: `yield`.

Пример:

```python
def count_to_three():
    yield 1
    yield 2
    yield 3
```

Использование:

```python
for number in count_to_three():
    print(number)
```

Результат:

```python
1
2
3
```

### Чем `yield` отличается от `return`

- `return` завершает функцию сразу
- `yield` отдает одно значение и "замораживает" функцию до следующего запроса

### Как это работает в твоем проекте

В файле `modules/list_initializers.py` есть генератор:

```python
def generate_random_numbers(length, left_border, right_border):
    for _ in range(length):
        yield round(uniform(left_border, right_border), 3)
```

Он не создает список целиком сразу, а выдает случайные числа по одному.

Потом здесь:

```python
list(generate_random_numbers(length, left_border, right_border))
```

генератор превращается в обычный список.

## 3. Что такое кортеж

Кортеж (`tuple`) — это коллекция, которую нельзя менять.

Пример:

```python
point = (3, 5)
```

В твоем проекте кортеж встречается здесь:

```python
return left_border, right_border
```

Это тоже кортеж.

Еще пример:

```python
result, elapsed_time = calculate_arccos_series(x_value, eps_value)
```

Слева идет распаковка кортежа на две переменные.

## 4. Что делает твой код по файлам

### `main.py`

Точка входа в программу.

Что происходит:

- импортируется `run_application`
- если файл запущен напрямую, стартует вся программа

### `modules/app.py`

Главный управляющий файл.

Он:

- показывает заголовок программы
- показывает меню
- читает выбор пользователя
- запускает нужную задачу
- спрашивает, продолжать ли работу

Это можно считать "диспетчером" программы.

### `modules/input_utils.py`

Файл для ввода данных.

Он:

- читает строку от пользователя
- преобразует ввод в `int` или `float`
- повторяет запрос, если пользователь ошибся
- проверяет специальные условия: например, что `x` в диапазоне `[-1, 1]`

Тут активно используется `try / except`, чтобы программа не падала от неверного ввода.

### `modules/validators.py`

Файл для проверок.

Он:

- превращает текст в число
- проверяет диапазон
- проверяет, что число положительное
- проверяет корректность границ интервала

Идея хорошая: ввод и проверка разделены по разным файлам.

### `modules/decorators.py`

Файл с декоратором.

Его задача:

- измерять время выполнения вычислительных функций

### `modules/ui.py`

Файл отвечает за вывод текста в консоль.

Он:

- печатает меню
- печатает заголовки задач
- красиво выводит результаты

То есть вычисления и оформление вывода у тебя разделены.

### `modules/series_math.py`

Математика для задачи 1.

Функция:

- считает `arccos(x)` через ряд
- сравнивает результат с `math.acos(x)`
- считает абсолютную ошибку
- использует `for ... else`

Почему здесь есть `for ... else`:

- `for` пытается найти решение за максимум 500 итераций
- если `break` сработал, значит нужная точность достигнута
- если `break` не сработал, выполняется `else` и выбрасывается ошибка

То есть здесь `for ... else` значит:
"если цикл не прервался раньше, значит мы не достигли цели".

### `modules/sequence_math.py`

Математика для задачи 2.

Функция:

- берет список целых чисел
- выбирает четные
- считает их среднее арифметическое

Если четных чисел нет, выбрасывается `ArithmeticError`.

### `modules/text_math.py`

Файл для работы с текстом.

Он умеет:

- считать пробелы и знаки пунктуации
- выделять слова
- считать частоту символов
- искать слова после запятых
- считать слова, которые начинаются или заканчиваются на гласную

### `modules/list_initializers.py`

Файл для создания списка.

Есть 2 режима:

- пользователь вводит элементы сам
- программа генерирует числа случайно

Именно здесь у тебя есть генератор.

### `modules/list_math.py`

Файл для задачи 5.

Он:

- находит отрицательные элементы
- считает их произведение
- находит максимум
- берет положительные элементы до максимума
- считает их сумму

### `modules/tasks/task1.py` ... `task5.py`

Это файлы-запускатели задач.

Каждый такой файл:

- выводит заголовок задачи
- получает данные
- вызывает нужную функцию из математического модуля
- обрабатывает возможные ошибки
- выводит результат

## 14. Удаление по значению и по индексу

### Удаление по значению: `remove()`

```python
numbers = [3, 5, 7, 5]
numbers.remove(5)
print(numbers)
```

Ответ:

```python
[3, 7, 5]
```

Важно:

- `remove(5)` удаляет первое найденное значение `5`
- удаление идет по значению, а не по номеру

### Удаление по индексу: `pop()`

```python
numbers = [3, 5, 7, 9]
numbers.pop(1)
print(numbers)
```

Ответ:

```python
[3, 7, 9]
```

Потому что удалился элемент с индексом `1`, то есть число `5`.

### Что будет с `5`

```python
numbers = [1, 5, 8]
numbers.remove(5)
print(numbers)
```

Ответ:

```python
[1, 8]
```

### Что будет с `-1`

```python
numbers = [1, 5, 8]
numbers.pop(-1)
print(numbers)
```

Ответ:

```python
[1, 5]
```

Потому что `-1` в `pop(-1)` означает последний индекс.

### Ошибка при `remove`

```python
numbers = [1, 2, 3]
numbers.remove(5)
```

Ответ:

Будет ошибка:

```python
ValueError
```

Потому что значения `5` в списке нет.
